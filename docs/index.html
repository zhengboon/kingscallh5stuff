Unified GUI Suite

All monitoring and orchestration tools combined into a single Python application.

Features

  • Game Launcher: Bootstrap 1–12 instances via Microsoft Edge or Chrome. Internet Explorer was retired by Microsoft on June 15, 2022 and no longer launches modern web apps.
  • Auto-Login: Generates ephemeral Chrome Extensions from accounts.txt with independent --user-data-dir profiles per instance.
  • Keyboard Macro Fallback: OS-level typing for 100% login success on protected pages.
  • Live Recorder: Background mss screenshots + JSONL event logs with disk space protection.
  • Status Dashboard & Feedback: Web dashboard featuring a Virtual Board matrix of the bot's inner state and a one-click feedback logging system.
  • APK Browser: Explore 215 decompiled game tables in a tree view.

Key Mechanics

  • 4 lanes × ~12-square deep battlefield grid; creatures advance and collide
  • 4 hero classes (Warrior, Ranger, Mage, Priest) with class-locked skill cards
  • 6-tier card rarity (Common → Godlike) plus Awakened upgrades
  • 33+ AOE geometry patterns
  • Dragon Forms: 5 wing + 5 scale variants
  • Race aura synergies across 7 base factions plus Outsiders, Dragons, Angels, Demons, and Hybrid
  • Full pet (launched Sept 2024), guild, and gacha systems

Modular Architecture

  • Vision: OpenCV template matching + edge density detection, MSS capture.
  • Engine: Deterministic game state with event bus, abilities, modifiers, combat resolver.
  • Agents: Heuristic priority rules + random baseline.
  • RL Env: Gymnasium-compliant wrapper for training.
  • Logging: Structured logging module throughout, JSONL session traces.
  • Recorder: Disk-space-aware background capture with auto-stop.

Engine Model — Simplification Note

The deterministic Python engine in cardbot/engine/ is an intentional simplification of the live game. It models lanes as one slot per side and resolves attacks directly to hero HP rather than simulating the 12-square corridor. This is sufficient for vision-driven mirroring, heuristic decision-making, and tabular Q-learning prototypes — not for ground-truth match simulation. The default board is 4 lanes (matching the live game); 2- and 3-lane modes are kept for unit-test ergonomics.
